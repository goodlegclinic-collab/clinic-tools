### 工具內容

貼上（或 🎤 語音輸入）病人主訴，自動計算：

| 分數 | 用途 |
|---|---|
| **VCSS**（0–30） | 靜脈臨床嚴重度，十項各 0–3，附證據反白、可手動覆寫 |
| **CEAP C 分級** | C0–C6（含 C4a/C4b），由主訴推定 |
| **VDS**（0–3） | 靜脈失能（工作能力影響） |
| **Villalta** | 血栓後症候群；勾「有 DVT 病史」才啟用 |
| **CIVIQ-20** | 生活品質（意譯參考版），GIS 0–100＋四面向 |

另含：門診快速五問、37 題完整問診總表、補問清單、一鍵複製病歷摘要、**自動產生主訴摘要**（50–100 字敘述＋自動附上全部分數，一鍵複製貼病歷）。

**回診模式**：勾「回診病人」→ 把上次病歷裡的主訴摘要（含分數）貼回來 → 顯示「回診快速七問」問變化 → 重新評分後自動產生**上次 → 本次比較表**（▼改善／▲惡化）與回診主訴（含 VCSS 12→8 箭號紀錄）。病歷即資料庫，工具本身不儲存任何病人資料。

## Google Drive 連動（5 分鐘設定）

設定完成後：填「病歷號」→ 評分 → 按「☁ 上傳 Drive」（或設成自動），系統就會：

1. 在你 Drive 的 **「VCSS追蹤總表」** 加一列：時間、病歷號、初診/回診、VCSS 總分＋十項逐項、CEAP、VDS、Villalta、CIVIQ、主訴摘要 → **統計/寫論文直接用這張表，不用回去翻病歷**
2. 在病人資料夾自動建立 `病歷號_日期_VCSS.txt`（主訴＋完整摘要）

資料只進**你自己的 Google 帳號**，不經過任何第三方伺服器。

### 部署步驟

1. 開 [script.google.com](https://script.google.com) → 「新增專案」
2. 刪掉預設內容，把本 repo 的 [`google-apps-script.gs`](google-apps-script.gs) 全部貼上
3. 改最上面兩行設定：
   - `TOKEN`：自訂一組密碼
   - `PARENT_FOLDER_ID`：病人資料夾網址 `folders/` 後面那串 ID（留空 = 只寫總表）
4. 右上「部署」→「新增部署作業」→ 類型選 **網頁應用程式** → 執行身分 **我** → 存取權 **知道連結的任何人** → 部署 → **複製網址**（`/exec` 結尾）
   - 第一次會要求授權：選你的帳號 → 進階 → 前往（這是授權「你自己的腳本」存取「你自己的 Drive」）
5. 回到評分工具 → 頁面底部「⚙ Google Drive 連動設定」→ 貼上網址與密鑰 → 儲存 → 填個測試病歷號評分一次、按「☁ 上傳 Drive」確認出現 ✓

> 「知道連結的任何人」指的是這支接收器網址；網址無法被猜到，且已設密鑰驗證，資料寫入只進你的 Drive。改過腳本後要「部署 → 管理部署作業 → 編輯 → 新版本」才會生效。

純前端運算，主訴內容不會上傳到任何伺服器（除非你自己設定 Google Drive 連動——那也只送進你自己的帳號）。臨床輔助初稿工具，不取代醫師診斷。

### 其他工具

- **[門診對話分析器](https://goodlegclinic-collab.github.io/clinic-tools/門診對話分析器.html)** — 貼上門診錄音文字稿（有無「醫師：／病人：」標記都可以，貼上約 1 秒自動分析），自動產出：
  - **對話總覽**：醫病發言佔比、提問句數、問答配對（醫師問「會痠嗎？」→ 病人答「會」也能正確計分；醫師的問句本身不計分，避免問診內容誤判成症狀）
  - **VCSS ＋ CEAP ＋ VDS 評分**（與評分工具同一套引擎，附證據反白、可覆寫）
  - **治療方案討論**（微波／EVLA／UFLA／膠水／硬化劑／DFPP／高壓氧…含費用討論）、**病人疑慮分類**（費用、怕痛、恢復期、保險、復發…附原句）、**醫囑・待辦・追蹤**與**決策狀態**（已約手術／考慮中／諮詢）
  - **對話品質計分卡（銷售／說服模型）**：以 SPIN Selling（需求探詢四類問句）、FAB（特色→好處→證據）、LAER（異議處理率）、AIDA（收尾行動）與 Cialdini 影響力六原則做規則式偵測，給 0–100 品質總分＋等級＋「給醫師的教練建議」（依對成交的影響排序、附具體話術）。醫療溝通以誠信與病人利益為前提，模型用於幫病人安心決策
  - **成交追蹤與成交率分析**：分析上傳帶成交狀態（追蹤中／當場成交／後續成交／未成交＋項目＋金額）；病人事後才決定 → 回工具填病歷號一鍵回填總表；「📈 成交率分析」讀取總表自動比對**成交 vs 未成交對話差在哪**（品質分、異議處理、SPIN、收尾），給數據洞察
  - **主訴摘要＋病歷摘要**一鍵複製（含品質分數與教練建議前三條）；回診模式自動比較上次分數
  - **Google Drive 連動**（與評分工具共用同一組設定）：填病歷號 → 「📂 抓 Drive 文字稿」列出病人資料夾裡的 .txt／Google 文件，點一下載入分析；資料夾裡有上次 VCSS／分析紀錄會自動帶入回診比較；「👁 自動監看」每 30 秒檢查一次，新文字稿一進 Drive 就自動分析；「☁ 上傳 Drive」寫入「門診對話分析總表」＋病人資料夾建檔
  - ※ 用 Drive 抓檔功能需以**最新版 `google-apps-script.gs`** 重新部署（部署 → 管理部署作業 → 編輯 → 新版本），第一次會再要求一次授權

- **[發票名目產生器](https://goodlegclinic-collab.github.io/clinic-tools/發票名目產生器.html)** — 開發票行政工具：一鍵複製「索取資訊」訊息 → 貼上對方回覆（表格式或口語都可）自動解析抬頭、統編、金額（自動判斷未稅／含稅並交叉驗算）、品名（依內容自動判斷，預設合作推廣費）、收件資訊 → 一鍵複製完整發票名目。純前端運算，資料不上傳。
