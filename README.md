### 工具內容

貼上（或 🎤 語音輸入）病人主訴，自動計算：

| 分數 | 用途 |
|---|---|
| **VCSS**（0–30） | 靜脈臨床嚴重度，十項各 0–3，附證據反白、可手動覆寫 |
| **CEAP C 分級** | C0–C6（含 C4a/C4b），由主訴推定 |
| **VDS**（0–3） | 靜脈失能（工作能力影響） |
| **Villalta** | 血栓後症候群；勾「有 DVT 病史」才啟用 |
| **CIVIQ-20** | 生活品質（意譯參考版），GIS 0–100＋四面向 |

另含：門診快速五問、37 題完整問診總表、補問清單、一鍵複製病歷摘要、**自動產生主訴摘要**（50–100 字敘述＋自動附上全部分數，一鍵複製貼病歷）。

**回診模式**：勾「回診病人」→ 把上次病歷裡的主訴摘要（含分數）貼回來 → 顯示「回診快速七問」問變化 → 重新評分後自動產生**上次 → 本次比較表**（▼改善／▲惡化）與回診主訴（含 VCSS 12→8 箭號紀錄）。病歷即資料庫，工具本身不儲存任何病人資料。

## Google Drive 連動（5 分鐘設定）

設定完成後：填「病歷號」→ 評分 → 按「☁ 上傳 Drive」（或設成自動），系統就會：

1. 在你 Drive 的 **「VCSS追蹤總表」** 加一列：時間、病歷號、初診/回診、VCSS 總分＋十項逐項、CEAP、VDS、Villalta、CIVIQ、主訴摘要 → **統計/寫論文直接用這張表，不用回去翻病歷**
2. 在病人資料夾自動建立 `病歷號_日期_VCSS.txt`（主訴＋完整摘要）

資料只進**你自己的 Google 帳號**，不經過任何第三方伺服器。

### 部署步驟

1. 開 [script.google.com](https://script.google.com) → 「新增專案」
2. 刪掉預設內容，把本 repo 的 [`google-apps-script.gs`](google-apps-script.gs) 全部貼上
3. 改最上面兩行設定：
   - `TOKEN`：自訂一組密碼
   - `PARENT_FOLDER_ID`：病人資料夾網址 `folders/` 後面那串 ID（留空 = 只寫總表）
4. 右上「部署」→「新增部署作業」→ 類型選 **網頁應用程式** → 執行身分 **我** → 存取權 **知道連結的任何人** → 部署 → **複製網址**（`/exec` 結尾）
   - 第一次會要求授權：選你的帳號 → 進階 → 前往（這是授權「你自己的腳本」存取「你自己的 Drive」）
5. 回到評分工具 → 頁面底部「⚙ Google Drive 連動設定」→ 貼上網址與密鑰 → 儲存 → 填個測試病歷號評分一次、按「☁ 上傳 Drive」確認出現 ✓

> 「知道連結的任何人」指的是這支接收器網址；網址無法被猜到，且已設密鑰驗證，資料寫入只進你的 Drive。改過腳本後要「部署 → 管理部署作業 → 編輯 → 新版本」才會生效。
